# RTSP Face Detection — FFmpeg GPU + TensorRT
**Author:** Anukool Kashyap  
**Repo:** [Benchmarking-Real-Time-Video-Analytics-Frameworks](https://github.com/AnukoolKashyap/Benchmarking-Real-Time-Video-Analytics-Frameworks)

RTSP live camera face detection using FFmpeg GPU decode (NVDEC via PyAV) + TensorRT YOLOv8-face inference. Built and tested on a 2560×1440 HEVC IP camera at 25 FPS, achieving ~27 FPS end-to-end with GPU decode + TensorRT inference.

## Actual Pipeline (as built and tested)

```
IP Camera (2560×1440, HEVC/H265, 25 FPS)
    ↓  RTSP over TCP
libavformat (PyAV av.open)
    — opens RTSP connection
    — reads compressed HEVC packets from network
    ↓  HEVC packets
libavcodec — hevc_cuvid decoder
    — routes HEVC packets to NVDEC hardware chip on GPU
    — decodes entirely on GPU, CPU does nothing here
    ↓  decoded frame in VRAM (NV12 format)
libswscale (inside PyAV to_ndarray)
    — NV12 → BGR24 conversion
    — frame transferred to CPU RAM as numpy array
    ↓  BGR numpy array (H, W, 3)
Preprocess (CPU)
    — letterbox resize to 640×640
    — normalize [0,255] → [0.0,1.0]
    — BGR → RGB, HWC → NCHW
    ↓  float32 tensor (1, 3, 640, 640)
TensorRT Engine — yolov8n-face.engine
    — FP32, TRT 11.1.0.106, CUDA 12.1
    — inference on GPU via execute_async_v3
    ↓  output tensor (1, 20, 8400)
Postprocess + NMS (CPU)
    — decode cx/cy/w/h/conf boxes
    — scale back to original resolution
    — OpenCV NMSBoxes removes duplicates
    ↓  list of (x1, y1, x2, y2, confidence)
FrameSaver
    — saves annotated JPEG + JSON metadata
    — to captured_faces/
```

## Setup

### Prerequisites

- NVIDIA GPU with NVDEC support
- CUDA 12.1
- Python 3.10
- Windows 10/11

### Step 1 — Install packages

```bash
pip install torch --index-url https://download.pytorch.org/whl/cu121
pip install av
pip install tensorrt-cu12 --extra-index-url https://pypi.nvidia.com --trusted-host pypi.nvidia.com
pip install pycuda
pip install ultralytics opencv-python numpy Pillow tqdm python-dotenv
```

### Step 2 — Set up environment variables

```bash
# copy .env.example to .env
copy .env.example .env
```

Edit `.env` and set your camera URL:
```
RTSP_URL=rtsp://USERNAME:PASSWORD@CAMERA_IP:PORT/stream_path
```

### Step 3 — Download model

Download `yolov8n-face.pt` from:
```
https://github.com/akanametov/yolo-face
```
Place in `models/` folder.

### Step 4 — Build TensorRT engine (run once on target GPU)

```bash
python build_engine.py
```

Takes 1–5 minutes. Generates `models/yolov8n-face.engine`.  
Note: engine file is GPU-specific — rebuild if switching machines.

### Step 5 — Check GPU setup

```bash
python scripts/check_gpu.py
```

All items should show ✓ before proceeding.

### Step 6 — Test camera connection

```bash
# CPU mode first — confirms camera is reachable
python scripts/test_camera.py --url "YOUR_RTSP_URL" --mode cpu

# GPU mode
python scripts/test_camera.py --url "YOUR_RTSP_URL" --mode gpu
```

### Step 7 — Run pipeline

```bash
python main.py
```

## Key Concepts

| Term | What it means |
|------|---------------|
| PyAV | Python wrapper around FFmpeg's libav libraries |
| libavformat | Opens RTSP stream, reads HEVC packets from network |
| libavcodec | Decodes HEVC — routes to NVDEC hardware when hevc_cuvid used |
| hevc_cuvid | Codec name that sends HEVC decode to NVDEC chip on GPU |
| NVDEC | Dedicated video decode chip on NVIDIA GPU — does not use CUDA cores |
| NV12 | Pixel format NVDEC outputs — a YUV variant, needs conversion to BGR |
| libswscale | Converts NV12 → BGR and handles resize |
| AVHWDeviceContext | CUDA bridge that lets libav and TensorRT share GPU memory |
| TensorRT | Compiles YOLO model to GPU-optimised engine file |
| execute_async_v3 | TensorRT 11 inference API (replaces execute_async_v2) |

## Difficulties Faced and How They Were Resolved

### 1. TensorRT not available via conda
**Problem:** `conda install tensorrt` failed with `PackagesNotFoundError` — NVIDIA never published TensorRT to conda channels.  
**Fix:** Installed via pip using NVIDIA's own PyPI index:
```bash
pip install tensorrt-cu12 --extra-index-url https://pypi.nvidia.com --trusted-host pypi.nvidia.com
```

### 2. SSL certificate error during pip install
**Problem:** `ssl.SSLError: [ASN1: NOT_ENOUGH_DATA]` when pip tried to download from `pypi.nvidia.com` — caused by a broken Windows certificate store in the conda environment.  
**Fix:** Used `--trusted-host` flag to bypass SSL verification for NVIDIA's host only.

### 3. trtexec not included in pip TensorRT
**Problem:** After installing TensorRT via pip, `trtexec.exe` was not available — it is only included in NVIDIA's full ZIP package, not the pip wheel.  
**Fix:** Built the engine using the Python TensorRT API directly (`build_engine.py`) instead of calling trtexec.

### 4. Multiple TensorRT 11 API breaking changes
**Problem:** All online examples and documentation use TensorRT 8/9 API. TRT 11 removed or renamed several key APIs causing `AttributeError` on every attempt.  
**Fixes applied:**

| Broken API | TRT 11 replacement |
|---|---|
| `EXPLICIT_BATCH` flag in `create_network()` | Removed — now default, pass no flags |
| `builder.platform_has_fast_fp16` | Removed — just set FP16 flag directly |
| `BuilderFlag.FP16` | Renamed — check available flags at runtime |
| `BuilderFlag` not iterable | Use try/except to probe flag names |
| `engine.get_binding_index()` | Replaced by `engine.get_tensor_name(i)` |
| `engine.binding_is_input()` | Replaced by `engine.get_tensor_mode(name)` |
| `engine.get_binding_shape()` | Replaced by `engine.get_tensor_shape(name)` |
| `engine.get_binding_dtype()` | Replaced by `engine.get_tensor_dtype(name)` |
| `context.execute_async_v2(bindings=...)` | Replaced by `set_tensor_address()` + `execute_async_v3()` |

### 5. GStreamer gi/PyGObject not available for Python 3.10
**Problem:** GStreamer is installed globally on Windows but its Python bindings (`gi` / PyGObject) were compiled for Python 3.9 (bundled with GStreamer) and Python 3.12 (MSYS2). No pre-built bindings exist for Python 3.10.  
**Fix:** Switched to PyAV (FFmpeg's libav) for RTSP capture instead of GStreamer. PyAV installs via pip with no system dependencies and provides equivalent GPU decode capability via hevc_cuvid.

### 6. Camera using HEVC (H265) not H264
**Problem:** Pipeline was configured for `h264_cuvid` but the camera streams HEVC/H265. This caused GPU decode to fail silently, returning 0–2 FPS.  
**Fix:** Changed codec to `hevc_cuvid` in config after detecting `codec: hevc` in the stream info.

### 7. RTSP URL had proto=OnVif suffix causing 0.6 FPS
**Problem:** The original RTSP URL included `&proto=OnVif` which forces ONVIF protocol. This caused extreme throttling — only 0.6 FPS despite the camera advertising 25 FPS.  
**Fix:** Removed `&proto=OnVif` from the URL. FPS immediately jumped to 23+ FPS.

### 8. PyAV codec_context.codec not writable
**Problem:** Attempted to override the GPU codec after opening the container via `video_stream.codec_context.codec = av.codec.Codec("hevc_cuvid", "r")` — PyAV does not allow modifying codec_context after stream is opened.  
**Fix:** Used a separate `CodecContext.create(gpu_codec)` approach to decode packets through a GPU context independently from the container's default context.

### 9. av.AVError does not exist in PyAV 12+
**Problem:** `except av.AVError` raised `AttributeError: module 'av' has no attribute 'AVError'` — the exception class was moved in newer PyAV versions.  
**Fix:** Changed to `except av.error.FFmpegError` which is the correct exception class in PyAV 12+.

### 10. RTSP URL contains special characters (# in password)
**Problem:** Camera password contained `#` which is a URL reserved character. Passing it raw in the URL caused authentication failures.  
**Fix:** URL-encoded the `#` as `%23` in the RTSP URL string.
